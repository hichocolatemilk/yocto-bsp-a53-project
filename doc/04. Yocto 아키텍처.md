# Yocto 아키텍처 이해 및 적용

## 1. 목표
- Yocto Project의 구조와 핵심 구성 요소를 이해하고,  
- Layer 기반으로 Linux 시스템을 구성하는 방법을 학습

---
## 2. 수행내용

### 1) Yocto 기본 개념
- Yocto는 여러 Layer를 조합하여 하나의 Linux 시스템을 구성하는 빌드 시스템.

#### 핵심 구성 요소
- **Layer**: 기능 단위 묶음 (패키지, BSP, 커널 등) ex) meta-custom-a53, meta-raspberrypi
- **Recipe (.bb)**: 패키지 빌드 방법 정의 ex) hello.bb
- **bbappend**: 기존 Recipe 확장  systemd_%.bbappend (고정IP 설정 추가)
- **Image Recipe**: root filesystem 구성 정의 ex) a53-image.bb (최종 타겟 이미지)
- **Machine**: 보드 설정 (BSP) ex) raspberrypi3-64 (Cortex-A53 아키텍처)
- **Distro**: 배포 정책 설정 ex) custom-a53.conf (전용 배포 정책 수립)
- **BitBake**: 빌드 엔진 ex) bitbake a53-image 명령을 통한 통합 빌드

### 2) 프로젝트 구조(요약)
- 본 프로젝트는 다음과 같은 구조로 구성하였다.
```Bash
yocto/
├── poky
|   └── build
├── meta-raspberrypi
├── meta-openembedded
├── meta-custom-a53
```
#### Layer 구성
- **poky**: Yocto 기본 빌드 시스템
- **meta-raspberrypi**: Raspberry Pi BSP
- **meta-openembedded**: 추가 패키지
- **meta-custom-a53**: 사용자 정의 Layer

### 4) Custom Layer 설계 
- 프로젝트 요구사항을 반영하기 위해 Custom Layer를 구성하였다.
```Bash
meta-custom-a53/
├── recipes-app
│ └── hello
├── recipes-core
│ └── images
│ └── a53-image.bb
├── recipes-kernel
│ └── linux
│ └── patch 적용
└── conf
└── distro
```
#### 주요 구현 내용
- custom image (a53-image) 생성
- application(hello) 추가
- systemd 네트워크 설정 추가
- kernel patch 적용
- device tree 수정

### 5) Recipe 구성 및 주요 변수
#### 기본 구조
주요 변수
경로 관련
WORKDIR: 빌드 작업 디렉토리
S: 소스 디렉토리
D: rootfs 설치 경로
bindir: 실행파일 (/usr/bin)
sysconfdir: 설정파일 (/etc)
이미지 설정
IMAGE_INSTALL += "hello"
IMAGE_FEATURES += "ssh-server-openssh"
파일 경로 확장
FILESEXTRAPATHS:prepend := "${THISDIR}/files:"
### 6) BitBake 빌드 흐름
- Yocto는 Task 기반으로 빌드가 진행된다.
do_fetch    → 소스 다운로드
do_unpack   → 압축 해제
do_patch    → 패치 적용
do_configure→ 설정
do_compile  → 컴파일
do_install  → rootfs 설치
