## 1. 목표
- 프로젝트에 적합한 Yocto 브랜치를 선정하여 안정적인 BSP 개발 환경 구성

---
## 2. 수행 내용
### 1) Yocto 브랜치 구조 이해
- Yocto는 약 6개월 단위로 릴리스가 업데이트됨
- 릴리스마다 지원 기간(EOL)이 존재하며, 장기 지원(LTS) 버전이 별도로 제공됨 

### 2) 브랜치 선택 기준 수립
- 최신 기능보다 안정성과 유지보수를 고려
- BSP 개발 특성상 **장기 지원(LTS) 버전**  선택

### 3) 브랜치 선택
- 최신 LTS 버전인 **Scarthgap (5.0 LTS)** 기준으로 환경 구성
- 장기 지원 기간을 고려하여 안정적인 개발 환경 확보 

### 4) 브랜치 git clone
-Quick Build에 따르면 Yocto는 bitbake-setup 툴을 이용하여 구성할 수도 있으나
본 프로젝트에서는 구조 이해를 위해 poky 저장소를 직접 clone하는 방식을 사용.
```Bash
cd /home/hello
mkdir yocto
cd yocto
# Poky 코어 저장소 클론 (Scarthgap 브랜치)
git clone -b scarthgap git://git.yoctoproject.org/poky
# Raspberry Pi BSP 레이어 클론 (Scarthgap 브랜치)
git clone -b scarthgap https://git.yoctoproject.org/meta-raspberrypi
```
