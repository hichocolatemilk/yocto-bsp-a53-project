# Linux Boot Flow & System Structure

## 1. 개요
- 임베디드 리눅스 시스템은 Bootloader, Kernel, RootFS로 구성되며,
- 각 단계는 순차적으로 실행되며 시스템을 초기화한다.

---

## 2. Boot Flow
### 1) 일반적인 Embedded Boot 구조 (AM335x 기준)
- SPL(MLO) → U-Boot → Kernel → RootFS
  
- **SPL (MLO)**
  - 최소한의 하드웨어 초기화
  - DDR 메모리 초기화
    
- **U-Boot**
  - 커널 이미지(zImage) 및 Device Tree(DTB)를 메모리에 로드
  - bootargs를 통해 커널 파라미터 전달

- **Kernel**
  - zImage 압축 해제
  - DTB 기반 하드웨어 초기화
  - 드라이버 로드

- **RootFS**
  - 사용자 공간 실행
  - systemd / init 프로세스 시작

---

### 2) Raspberry Pi Boot 구조
- GPU Boot → config.txt → cmdline.txt → Kernel → RootFS
- **bootcode.bin / start.elf**
  - GPU에서 하드웨어 초기화 수행
  - Kernel 이미지 로드

- **config.txt**
  - 하드웨어 설정 (GPU/CPU, 메모리, UART, HDMI 등)
  - Device Tree Overlay 설정

- **cmdline.txt**
  - 커널 부팅 파라미터 전달
  - rootfs 위치, console 설정 등

- **Kernel**
  - DTB 기반으로 하드웨어 초기화 및 드라이버 로드

- **RootFS**
  - 사용자 공간 실행

---

### 3) U-Boot 기반 Boot 구조
- BootROM → U-Boot → Kernel → RootFS

- **BootROM**
  - 기본 하드웨어 초기화 수행

- **U-Boot**
  - Kernel(zImage) 및 DTB 로드
  - bootargs를 통해 커널 파라미터 전달

- **Kernel**
  - 하드웨어 초기화 및 드라이버 로드

- **RootFS**
  - systemd 실행 및 사용자 공간 시작

---

## 3. 주요 설정 파일

### 1) config.txt
- 위치: `/boot/config.txt`
- 역할: 하드웨어 설정

#### 예시
- GPU 메모리 설정
- HDMI 설정
- UART 활성화
- Device Tree Overlay 적용

---

### 2) cmdline.txt
- 위치: `/boot/cmdline.txt`
- 역할: 커널 부팅 파라미터 전달

#### 주요 옵션
- `root=/dev/mmcblk0p2` → root filesystem 위치
- `console=ttyS0,115200` → 콘솔 출력 설정
- `quiet` → 부팅 로그 최소화

---

## 4. 정리
- Bootloader는 Kernel을 메모리에 로드하고 실행을 넘김
- Kernel은 하드웨어 초기화 및 드라이버를 담당
- RootFS는 사용자 공간 실행
