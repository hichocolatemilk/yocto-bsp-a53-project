 CMake 기반 빌드 시스템 구성

## 1. 목표
- OpenSSL 및 네트워크 코드가 포함된 프로젝트를  
- 효율적으로 빌드할 수 있도록 CMake 기반 구조 구성  
- 라이브러리 의존성(OpenSSL, pthread) 자동 처리  
- 확장 가능한 빌드 환경 설계  

---

## 2. 수행 내용
### 1) CMake 사용 이유
- gcc 단일 명령으로는 관리 어려움
- 소스 파일 증가 시 유지보수 불편
- OpenSSL 링크 옵션이 복잡

### 2) CMakeLists.txt 작성
```Bash
cmake_minimum_required(VERSION 3.10)
project(openSSLProject)
```
#### 1. 헤더 파일 경로
```Bash
include_directories(include)
```
#### 2. OpenSSL 라이브러리 찾기
```Bash
find_package(OpenSSL REQUIRED)
```
#### 3. 실행 파일 생성
```Bash
add_executable(tcp_secure_client 
 src/socket/tcp/tcpClientThread.c 
 src/openssl/AES256-CBC_TEST.c
 src/openssl/sha256_TEST.c
)
```
#### 4. 라이브러리 링크
```Bash
target_link_libraries(tcp_secure_client 
 PRIVATE 
 OpenSSL::Crypto 
 pthread
)
```
