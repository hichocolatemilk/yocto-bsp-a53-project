# Yocto 빌드 방법

## 1. 프로젝트 목표
- 커스텀 Yocto 이미지를 빌드하고, 실제 하드웨어 타겟에 플래싱하기 위한 환경 구축 및 실행 절차.

---
## 2. 수행 내용
### 1) 빌드 환경 초기화
-Yocto 빌드 엔진인 `bitbake` 및 관련 도구들을 사용하기 위해 환경 변수를 설정.
-**주의**: 새로운 터미널 세션을 열 때마다 반드시 수행해야함.
```Bash
cd poky
source oe-init-build-env
```

### 2) 레이어 구성 및 검증
- 타겟 하드웨어 제어를 위한 BSP 레이어와 본 프로젝트의 핵심인 커스텀 레이어를 추가.
- 레이어 추가 후 show-layers 명령을 통해 빌드 시스템에 정상적으로 오버레이되었는지 확인
```Bash
bitbake-layers add-layer ../../meta-raspberrypi
bitbake-layers add-layer ../../meta-custom-a53
```

#### 등록된 레이어 리스트 확인
bitbake-layers show-layers
### 3) 타겟 머신 설정 및 distro(배포판) 설정
- build/conf/local.conf 파일 내에 Raspberry Pi 3B+ 64비트, distro(배포판) 선언.
#### build/conf/local.conf 설정 추가
MACHINE = "raspberrypi3-64"
DISTRO = "custom-a53"

### 4) 이미지 빌드 (Build Process)
본 프로젝트의 최종 산출물인 커스텀 이미지 a53-image를 빌드.
```Bash
bitbake a53-image
```

### 5) 산출물 확인 및 배포
-빌드 완료 후 생성된 이미지를 확인하고 실제 SD 카드에 플래싱합니다.
-이미지 경로: tmp/deploy/images/raspberrypi3-64/
-배포 파일: a53-image-raspberrypi3-64.rootfs.wic
-방법: 생성된 .wic 파일을 호스트 PC로 추출(SCP 등)한 뒤, Raspberry Pi Imager를 사용하여 SD 카드에 작성
