# U-Boot Environment 분석

## 1. 개요
U-Boot에서 `printenv` 명령을 통해  
부팅 과정에서 사용되는 주요 환경 변수를 분석.

---

## 2. 주요 환경 변수
### bootargs
- 커널에 전달되는 부팅 파라미터
#### 예시
```Bash
console=ttyS0,115200 root=/dev/mmcblk0p2 rootwait
```
#### 역할
- 콘솔 출력 설정
- root filesystem 위치 지정
- *bootcmd*
- U-Boot가 부팅 시 실행하는 명령
- *bootflow scan*
  
#### 역할
- 부팅 가능한 디바이스 탐색 후 커널 실행
- *fdtfile*
- Device Tree 파일 지정
- *broadcom/bcm2837-rpi-3-b-plus.dtb*

#### 역할
- 커널이 사용할 하드웨어 정보 제공
- *kernel_addr_r / loadaddr*
- 커널이 로드되는 메모리 주소
- 커널 이미지가 RAM에 로드되는 위치 지정
  
## 3. 정리
- U-Boot는 환경 변수를 기반으로 부팅을 수행함
- bootargs를 통해 커널 파라미터 전달
- bootcmd를 통해 부팅 흐름 제어
- fdtfile을 통해 하드웨어 정보 전달
