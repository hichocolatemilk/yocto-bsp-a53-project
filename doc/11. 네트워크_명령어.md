# Linux Network Configuration & Diagnostic

## 1. 목표
- 리눅스 시스템의 네트워크 인터페이스 상태를 확인하고, 라우팅 경로 및 ARP 테이블을 관리하여 통신 연결성을 검증하고 제어함.
  
---

## 2. 수행 내용
### 1) 인터페이스 정보 확인 (ifconfig & ip addr)
- 시스템에 할당된 IP 주소와 네트워크 인터페이스의 활성화 상태를 확인합니다.

```Bash
# 전통적인 방식 (net-tools 패키지 필요)
ifconfig
```
#### 현대적 표준 방식 (상세한 멀티 IP 확인 가능)
```Bash
ip addr show
# 차이점: ifconfig는 직관적이지만 하나의 인터페이스에 할당된 보조 IP를 생략, ip addr은 모든 할당 정보를 상세히 출력함.
```

### 2) 라우팅 테이블 및 게이트웨이 설정 (ip route)
- 데이터가 외부 망으로 나가는 경로(Route)를 확인하고 수정합니다.

```Bash
# 현재 라우팅 테이블 확인
ip route show

# 기본 게이트웨이(Default Gateway) 추가
ip route add default via 192.168.1.1 dev eth0

# 특정 네트워크 대역에 대한 정적 경로(Static Route) 추가
ip route add 10.10.10.0/24 via 192.168.1.254 dev eth1
```
### 3) ARP 캐시 테이블 관리 (arp)
- IP 주소를 물리적 MAC 주소로 매핑하는 테이블을 관리하여 로컬 통신 장애를 진단합니다.

```Bash
# 전체 ARP 테이블 확인
arp -a

# 특정 IP에 대한 ARP 정보 강제 삭제 (네트워크 충돌 해결 시 활용)
arp -d 192.168.1.100
```
