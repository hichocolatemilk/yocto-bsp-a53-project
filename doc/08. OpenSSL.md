# OpenSSL 기반 AES 암호화 및 해시 구현

## 1. 목표
- 임베디드 리눅스 환경에서 OpenSSL 라이브러리를 활용하여  
- AES-256-CBC 기반 암호화/복호화 구현  
- SHA-256 해시 생성 구현  
- TCP 통신에 적용 가능한 보안 구조 설계  

---

## 2. 수행 내용

### 1) OpenSSL 환경 확인
- OpenSSL 헤더 및 라이브러리 확인
- EVP API 사용 가능 여부 확인
- SDK에 OpenSSL 포함 확인
```bash
source /opt/custom-a53/1.0/environment-setup-cortexa53-poky-linux
ls $SDKTARGETSYSROOT/usr/include/openssl | grep evp.h
ls $SDKTARGETSYSROOT/usr/lib | grep libcrypto
```

### 2) Yocto 이미지에 OpenSSL 추가
- IMAGE_INSTALL:append = " openssl openssl-bin"
- 커스텀 이미지에 OpenSSL 패키지 포함
- CLI 및 라이브러리 사용 가능하도록 구성

### 3) OpenSSL CLI 테스트
- AES 암호화 결과 확인
- SHA256 해시 결과 확인
```Bash
echo -n "Hello World" | openssl enc -aes-256-cbc -K <key> -iv <iv> | hexdump -C
echo -n "Hello World" | openssl dgst -sha256
```

### 4) AES-256-CBC 암호화 구현
🔐 Key / IV 설정
unsigned char key[32] = { ... };
unsigned char iv[16] = { ... };
🔄 암호화 흐름
EVP_CIPHER_CTX_new()
→ EVP_EncryptInit_ex()
→ EVP_EncryptUpdate()
→ EVP_EncryptFinal_ex()
🔄 복호화 흐름
EVP_CIPHER_CTX_new()
→ EVP_DecryptInit_ex()
→ EVP_DecryptUpdate()
→ EVP_DecryptFinal_ex()

👉 EVP API 기반 구조로 구현

### 5) SHA-256 해시 구현
EVP_MD_CTX_new()
→ EVP_DigestInit_ex()
→ EVP_DigestUpdate()
→ EVP_DigestFinal_ex()
입력 데이터 → 고정 길이 해시 생성

👉 데이터 무결성 검증에 활용 가능


### 6) TCP 통신에 암호화 적용
송신:
평문 → AES 암호화 → send()
수신:
recv() → AES 복호화 → 출력

👉 네트워크 계층 위에 보안 계층 추가
