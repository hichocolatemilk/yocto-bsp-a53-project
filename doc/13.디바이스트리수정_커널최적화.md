# Kernel Optimization & Device Tree Customization

## 1. 목표
- 라즈베리파이 3B+(Cortex-A53) 하드웨어 자원을 효율적으로 관리하기 위해 Device Tree를 수정하고, 불필요한 기능을 제거하여 커널 부팅 속도 및 시스템 리소스를 최적화함.
  
---

## 2. 수행 내용
### 1) Device Tree (DTS) 기반 UART 활성화
- 기본 설정에서 제한된 시리얼 통신 기능을 확장하기 위해 디바이스 트리 소스를 수정함.
- 수정 대상: arch/arm/boot/dts/bcm2710-rpi-3-b-plus.dts (또는 관련 dtsi 파일)

#### 핵심 작업:
- UART1 포트의 핀 설정을 Alt5 모드로 변경하여 전용 시리얼 통신 라인을 확보함.
- local.conf에서 ENABLE_UART="1"을 해줘야 시리얼이 활성화가됨(라즈베리파이)

### 2) 시스템 상태 시각화 (Heartbeat LED 패치)
- 커널이 정상적으로 동작 중임을 외부에서 확인할 수 있도록 온보드 LED 설정을 변경함.
- Trigger 설정: LED 트리거를 heartbeat로 지정하여 커널 로드에 따라 LED가 점멸하도록 구현함.
**효과**: 디스플레이 장치가 없는 임베디드 환경에서 커널의 생존 여부(Kernel Alive)를 즉시 파악 가능함.

### 3) 커널 기능 최적화 (Menuconfig 활용)
- 불필요한 커널 드라이버 및 모듈을 제거하여 가벼운 커널 이미지를 생성함.

#### 최적화 항목:
- 현재 프로젝트에서 사용하지 않는 사운드 드라이버, 와이파이 스택 등을 제거.
- 부팅 시 출력되는 로고나 불필요한 디버그 옵션을 해제하여 부팅 시간을 단축함.

3. 결과 및 기대효과
- 커널 경량화를 통해 시스템 부팅 속도 개선
- Device Tree 기반 하드웨어 제어 방식 이해
- UART 및 LED 제어를 통해 임베디드 디버깅 환경 구축
