# Custom Layer 설계 및 레이어 확장

## 1. 목표
- Yocto 환경에서 Custom Layer를 분리하여  
- BSP 및 기능을 독립적으로 관리하고 확장 가능한 구조를 구성

---
## 2. 수행내용 
### 1)Custom Layer 생성
- Yocto 프로젝트 내에서 사용자 정의 Layer를 생성하였다.(old)
```bash
mkdir -p meta-custom-a53
```
- Yocto 프로젝트 내에서 사용자 정의 Layer를 생성하였다(new)
```basah
# yocto 디렉토리에서 해야함 build아님
bitbakelayer create-layer meta-custom-a53
```

### 2). Layer 구조 설계
- Custom Layer는 기능별로 다음과 같이 구성하였다.
```Bash
meta-custom-a53/
├── conf
│   ├── distro
│   │   └── custom-a53.conf
│	├── machine
|	|	 ├── custom-a53-6.6.conf
|   | 	 └── custom-a53-6.12.conf
│   └── layer.conf
├── recipes-app
│   └── hello
│       ├── files
│       │   ├── CMakeLists.txt
│       │   ├── hello.c
│       │   └── hello.service
│       └── hello.bb
├── recipes-core
│   ├── images
│   │   └── a53-image.bb
│    └── systemd
│        ├── files
│        │   └── 10-eth0.network
│        └── systemd_%.bbappend
├── recipes-kernel
│   └── linux
└── recipes-kernel
		    └── linux
		        ├── linux-raspberrypi
		        │   ├── 0001-bcm2710-rpi3b-uart0-enable-lec_act-heartbeat.patch
		        │   ├── 0002-dts-change-serial0-to-uart0-set-pins-to-alt5.patch
		        │   └── devtool-fragment.cfg
		        └── linux-raspberrypi_%.bbappend
```
### 3) Distro 설정 (custom-a53.conf)
- Custom Distro를 정의하여 시스템 정책을 설정하였다.
```Basah
require conf/distro/poky.conf
DISTRO = "custom-a53"
DISTRO_NAME = "Custom A53 Distro"
DISTRO_VERSION = "1.0"
LICENSE_FLAGS_ACCEPTED = "synaptics-killswitch"

RPI_USE_U_BOOT = "1"
ENABLE_UART = "1"
UBOOT_DELAY = "5"

DISTRO_FEATURES:append = " systemd usrmerge u-boot"
DISTRO_FEATURES_BACKFILL_CONSIDERED = "sysvinit"

VIRTUAL-RUNTIME_init_manager = "systemd"
VIRTUAL-RUNTIME_initscripts = ""
```
#### 적용 내용
- systemd 기반 init 시스템 적용
- sysvinit 제거
- U-Boot 사용 설정
- UART 활성화

### 4) Layer 등록 설정 (layer.conf)
- Custom Layer가 BitBake에서 인식되도록 설정.
```Bash
BBPATH .= ":${LAYERDIR}"
BBFILES += "${LAYERDIR}/recipes-*/*/*.bb \
            ${LAYERDIR}/recipes-*/*/*.bbappend"

BBFILE_COLLECTIONS += "customa53"
BBFILE_PATTERN_customa53 := "^${LAYERDIR}/"
BBFILE_PRIORITY_customa53 = "12"
LAYERSERIES_COMPAT_customa53 = "scarthgap"
```

### 5) Custom Image 구성 (a53-image)
- 필요한 패키지를 포함한 Custom Image를 구성하였다.
```Bash
IMAGE_INSTALL:append = " \
hello net-tools openssl tcpdump \
htop memtester vim iproute2 devmem2 gdb strace \
"
IMAGE_FEATURES += "ssh-server-dropbear"
inherit core-image
```
#### 포함 기능
- 네트워크 디버깅: tcpdump, iproute2
- 개발 도구: gdb, strace
- 시스템 분석: htop, memtester
- 보안 통신: openssl

### Layer 추가 및 의존성 해결
- 패키지 사용을 위해 meta-openembedded Layer를 추가하였다.
```Bash
cd /yocto-a53-project
git clone -b scarthgap git://git.openembedded.org/meta-openembedded
#yocto-a53-project/poky/build 에서 진행함 
bitbake-layers add-layer ../../meta-openembedded/meta-oe
bitbake-layers add-layer ../../meta-openembedded/meta-python
bitbake-layers add-layer ../../meta-openembedded/meta-networking
```

### 머신 설정(선택사항)
- 원래 커널버전은 scarthgap은 6.6이나 최신커널 6.12도 있길래 테스트
```bash
#/conf/machine/custom-a53-6.12.conf 생성
require conf/machine/raspberrypi3-64.conf # 라즈베리파이3-64를 상속
PREFERRED_VERSION_linux-raspberrypi = "6.12.%" # 버전 명시
KBUILD_DEFCONFIG:custom-a53 = "bcm2711_defconfig" # 없으면 오류가 나옴 defconfig를 못찾는듯 함
IMAGE_ROOTFS_EXTRA_SPACE = "4194304" #5GB로 늘림
```

### 문제
- Custom Image 빌드 시 다음과 같은 오류 발생:
Nothing RPROVIDES 'linux-firmware-rpidistro-bcm43455'

**원인**:
특정 firmware 패키지가 라이선스 제한으로 제외됨
해결
```Bash
LICENSE_FLAGS_ACCEPTED = "synaptics-killswitch"
```
