# TCP/UDP 소켓 통신 및 멀티스레드 구조

## 1. 목표
- 임베디드 리눅스 환경에서  
- TCP/UDP 기반 네트워크 통신을 구현하고,  
- 멀티스레드를 활용하여  
- 다중 클라이언트를 처리할 수 있는 구조를 설계

---

## 2. 수행 내용
### 1) UDP 통신 구현 (비연결형)
- socket()으로 UDP 소켓 생성
- sendto / recvfrom 기반 통신 구현

```c
sockfd = socket(AF_INET, SOCK_DGRAM, 0);
sendto(sockfd, message, len, 0, ...);
recvfrom(sockfd, buffer, BUFFER_SIZE, 0, ...);
sockaddr_in 구조체를 사용하여 IP/PORT 설정
포트 설정 시 htons() 적용
```
👉 UDP는 연결 없이 데이터 송수신 수행


### 2) TCP 통신 구현 (연결형)
- 서버 흐름
socket → bind → listen → accept → recv/send
- 클라이언트 흐름
socket → connect → send/recv
listen()으로 연결 대기 상태 구성
accept()로 클라이언트 연결 수립
recv / send를 통해 데이터 송수신

👉 TCP는 연결 기반 통신으로 신뢰성 보장


### 3) TCP 핵심 구조 이해 
listen_fd: 연결 요청 대기용 소켓
client_fd: 실제 통신용 소켓

👉 accept() 이후 생성된 소켓으로 통신 수행

### 4) 멀티스레드 기반 구조 구현
클라이언트 연결 시 thread 생성
pthread_create(&thread_id, NULL, client_handler, &client_sock);
서버:
클라이언트마다 독립 thread 생성
클라이언트:
recv 전용 thread + send 메인 thread

👉 다중 클라이언트 동시 처리 가능

### 5) Non-blocking 처리 (UDP)
setblocking(False) 또는 예외 처리 사용
sock.setblocking(False)
BlockingIOError 발생 시 continue 처리

👉 블로킹으로 인한 멈춤 방지


### 6) 문자열 처리 문제 해결
recv 후 문자열 끝 처리 필요
buffer[len] = '\0';

👉 미처리 시 문자열 깨짐 발생

### 7) 프로세스 vs 스레드 비교
구분	프로세스	스레드
메모리	독립	공유
생성 속도	느림	빠름
통신	복잡	쉬움

👉 스레드를 사용하여 성능 및 구현 효율 개선

## 3. 결과
UDP/TCP 통신 프로그램 구현 완료
멀티스레드 기반 확장 가능한 구조 설계
이후 OpenSSL 적용을 통한 보안 통신 확장 기반 확보
## 4. 핵심 포인트
TCP는 “listen / accept 구조 이해”가 핵심
UDP는 구조 단순하지만 신뢰성 없음
멀티스레드를 통해 실무형 구조로 확장
문자열 처리 및 블로킹 이슈 해결 경험 확보
