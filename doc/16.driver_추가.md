# Custom Character Device Driver Implementation

## 1. 목표
사용자 공간(User Space)과 커널 공간(Kernel Space) 간의 데이터 전송 메커니즘을 이해하고, 실제 하드웨어 제어를 위한 기초적인 캐릭터 디바이스 드라이버를 설계 및 통합함.
---
## 2. 수행 내용
- 캐릭터 디바이스 드라이버 설계:
- /dev/chartest 디바이스 노드를 생성하고 file_operations 구조체를 통해 open, read, write, release 함수를 구현함.
- copy_to_user 및 copy_from_user 함수를 사용하여 유저 영역과의 안전한 데이터 교환 로직을 이해.
- 자동으로 dev가 생기게 해주는 create_class, device_create 드라이버 종료시 dev를 자동으로 없애주는 class_destroy, destory_device 함수 이해  

### Yocto 프로젝트 기반 드라이버 통합:
- recipes-kernel 내에 커널 모듈 레시피를 생성하고, bitbake를 통해 이미지 빌드 시 자동 포함되도록 구성함.
- KERNEL_MODULE_AUTOLOAD 설정을 통해 부팅 시 드라이버가 자동으로 로드되도록 함.

### 3. 트러블슈팅 및 해결
#### 이슈 1: Yocto 빌드 환경 내 Makefile 경로 참조 오류
- **현상**:  드라이버를 할때 기존 TEST할떄는 all,clean만 했었으나 modules_install이 없어서 문제가 됨.
- **해결** :
```Bash
# Makefile에 추가
  modules_install:
	$(MAKE) -C $(KERNEL_SRC) M=$(SRC) modules_install INSTALL_MOD_PATH=$(DESTDIR) 
```

#### 이슈 2: Boundary Check 미비로 인한 커널 워닝(Buffer Overflow) 해결
- **현상**: cat /dev/chartest 실행 시 Buffer overflow detected 워닝과 함께 Bad address(-EFAULT) 에러 발생
- **원인**: 파일 위치 지시자(loff_t *off) 제어 누락 및 유저 요청 크기가 고정된 커널 버퍼(100 bytes)를 초과하여 발생함.
- **해결**: *off를 활용한 데이터 읽기 위치 추적 로직과 요청 크기를 제한하는 경계 검사(Boundary Check) 로직을 추가하여 시스템 안정성을 확보함.
